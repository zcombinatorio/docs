## Overview

Adds a wallet address to the DAO's proposer whitelist. Whitelisted wallets can create futarchy proposals without needing to meet the token threshold requirement.

<CodeGroup>

```bash curl
curl -X POST https://api.zcombinator.io/dao/Dao1xgp...ABC/proposers \
  -H "Content-Type: application/json" \
  -H "x-wallet-address: YOUR_WALLET" \
  -H "x-signature: YOUR_SIGNATURE" \
  -H "x-timestamp: 1736789200" \
  -d '{
    "wallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "proposer_wallet": "NewProposer1xgp...XYZ"
  }'
```


## Path Parameters

<ParamField path="daoPda" type="string" required>
  The DAO's Program Derived Address
</ParamField>

## Request Parameters

<ParamField body="wallet" type="string" required>
  The DAO owner's wallet address (must match x-wallet-address header)
</ParamField>

<ParamField body="proposer_wallet" type="string" required>
  The wallet address to add to the proposer whitelist
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the operation succeeded
</ResponseField>

<ResponseField name="proposer" type="object">
  The created proposer record
</ResponseField>

<ResponseField name="proposer.id" type="number">
  Internal proposer ID
</ResponseField>

<ResponseField name="proposer.dao_id" type="number">
  The DAO's internal ID
</ResponseField>

<ResponseField name="proposer.proposer_wallet" type="string">
  The whitelisted wallet address
</ResponseField>

<ResponseField name="proposer.added_by" type="string">
  The wallet that added this proposer
</ResponseField>

<ResponseField name="proposer.created_at" type="string">
  ISO timestamp of when the proposer was added
</ResponseField>

## Notes

- The DAO creator is automatically added as a proposer when the DAO is created
- Adding an already-whitelisted wallet will create a duplicate entry (consider checking first)
- Whitelisted proposers can create proposals regardless of token holdings
