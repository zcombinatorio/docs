## Overview

Creates a new parent DAO linked to an existing Meteora liquidity pool. The DAO creator becomes the owner and is automatically added to the proposer whitelist.

<CodeGroup>

```bash curl
curl -X POST https://api.zcombinator.io/dao/parent \
  -H "Content-Type: application/json" \
  -H "x-wallet-address: YOUR_WALLET" \
  -H "x-signature: YOUR_SIGNATURE" \
  -H "x-timestamp: 1736789200" \
  -d '{
    "wallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "name": "MyDAO",
    "token_mint": "ZComb1xgp...ABC",
    "treasury_cosigner": "Cosigner1xgp...XYZ",
    "pool_address": "Pool1xgp...123",
    "funding_signature": "5K7x..."
  }'
```

## Request Parameters

<ParamField body="wallet" type="string" required>
  The wallet address creating the DAO (must match x-wallet-address header)
</ParamField>

<ParamField body="name" type="string" required>
  DAO name (max 32 characters). Must be unique across all DAOs.
</ParamField>

<ParamField body="token_mint" type="string" required>
  Base token mint address (the project token). Must be an SPL Token (Token-2022 not yet supported).
</ParamField>

<ParamField body="treasury_cosigner" type="string" required>
  Squads multisig cosigner for treasury operations
</ParamField>

<ParamField body="pool_address" type="string" required>
  Meteora liquidity pool address containing the token pair
</ParamField>

<ParamField body="funding_signature" type="string" required>
  Transaction signature of the SOL funding transaction for DAO creation fees
</ParamField>

<ParamField body="api_key" type="string">
  Optional API key. If it matches the server's `DAO_RESERVE_API_KEY`, the DAO is created with visibility 3 (client test) instead of the default 0 (hidden).
</ParamField>

## Response

<ResponseField name="dao_pda" type="string">
  The on-chain DAO Program Derived Address
</ResponseField>

<ResponseField name="moderator_pda" type="string">
  The moderator PDA for proposal management
</ResponseField>

<ResponseField name="treasury_vault" type="string">
  The Squads treasury vault address
</ResponseField>

<ResponseField name="mint_vault" type="string">
  The Squads mint authority vault address
</ResponseField>

<ResponseField name="admin_wallet" type="string">
  The managed admin wallet for on-chain operations
</ResponseField>

<ResponseField name="pool_type" type="string">
  Detected pool type (`dlmm` or `cpAmm`)
</ResponseField>

<ResponseField name="quote_mint" type="string">
  The quote token mint derived from the pool
</ResponseField>

<ResponseField name="transaction" type="string">
  The on-chain transaction signature
</ResponseField>

## Creating a Child DAO

To create a child DAO under an existing parent, use `POST /dao/child`:

```bash
curl -X POST https://api.zcombinator.io/dao/child \
  -H "Content-Type: application/json" \
  -H "x-wallet-address: YOUR_WALLET" \
  -H "x-signature: YOUR_SIGNATURE" \
  -H "x-timestamp: 1736789200" \
  -d '{
    "wallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "name": "MyChildDAO",
    "parent_pda": "Dao1xgp...ABC",
    "token_mint": "ChildToken1xgp...XYZ",
    "treasury_cosigner": "Cosigner1xgp...XYZ",
    "funding_signature": "5K7x..."
  }'
```

Child DAOs inherit the parent's liquidity pool and can only be created by the parent DAO owner.
