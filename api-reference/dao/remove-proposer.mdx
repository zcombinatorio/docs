---
title: "Remove Proposer"
description: "DELETE /dao/:daoPda/proposers/:proposerWallet - Remove a wallet from the proposer whitelist"
---

## Overview

Removes a wallet address from the DAO's proposer whitelist. The wallet will no longer be able to create proposals unless they meet the token threshold requirement.

<CodeGroup>

```bash curl
curl -X DELETE https://api.zcombinator.io/dao/Dao1xgp...ABC/proposers/ProposerWallet1xgp...XYZ \
  -H "Content-Type: application/json" \
  -H "x-wallet-address: YOUR_WALLET" \
  -H "x-signature: YOUR_SIGNATURE" \
  -H "x-timestamp: 1736789200" \
  -d '{
    "wallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"
  }'
```

```javascript fetch
const daoPda = 'Dao1xgp...ABC';

const response = await fetch(

  {
    method: 'DELETE',
```

## Path Parameters

<ParamField path="daoPda" type="string" required>
  The DAO's Program Derived Address
</ParamField>

<ParamField path="proposerWallet" type="string" required>
  The wallet address to remove from the whitelist
</ParamField>

## Request Parameters

<ParamField body="wallet" type="string" required>
  The DAO owner's wallet address (must match x-wallet-address header)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the operation succeeded
</ResponseField>

<ResponseField name="removed_wallet" type="string">
  The wallet address that was removed
</ResponseField>

### Success Response

```json
{
  "success": true,
  "removed_wallet": "ProposerWallet1xgp...XYZ"
}
```

## Notes

- Removing the DAO owner from the whitelist is allowed but not recommended
- If the removed wallet holds enough tokens to meet the threshold, they can still create proposals
- Active proposals created by a removed proposer are not affected
