## Overview

Configures proposer eligibility requirements for a DAO. You can set a minimum token balance threshold and optionally require that the balance be held for a minimum period (time-weighted average).

Only fields present in the request body are updated â€” omitted fields are preserved.

<CodeGroup>

```bash curl
curl -X PUT https://api.zcombinator.io/dao/Dao1xgp...ABC/proposer-config \
  -H "Content-Type: application/json" \
  -H "x-wallet-address: YOUR_WALLET" \
  -H "x-signature: YOUR_SIGNATURE" \
  -H "x-timestamp: 1736789200" \
  -d '{
    "wallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "threshold": "1000000000",
    "holding_period_hours": 24
  }'
```

## Path Parameters

<ParamField path="daoPda" type="string" required>
  The DAO's Program Derived Address
</ParamField>

## Request Parameters

<ParamField body="wallet" type="string" required>
  The DAO owner's wallet address (must match x-wallet-address header)
</ParamField>

<ParamField body="threshold" type="string">
  Minimum token balance in **raw units** (including decimals). Set to `null`, `""`, or `"0"` to disable threshold-based access. Only updated if present in the request body.
</ParamField>

<ParamField body="holding_period_hours" type="number">
  Number of hours over which to calculate the time-weighted average balance. Must be between 1 and 35,040 (4 years). Set to `null` to disable. Cannot be set without a token threshold. Only updated if present in the request body.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the operation succeeded
</ResponseField>

<ResponseField name="dao_pda" type="string">
  The DAO's Program Derived Address
</ResponseField>

<ResponseField name="dao_name" type="string">
  The DAO's name
</ResponseField>

<ResponseField name="proposer_token_threshold" type="string | null">
  The current threshold value, or `null` if disabled
</ResponseField>

<ResponseField name="proposer_holding_period_hours" type="number | null">
  The current holding period in hours, or `null` if disabled
</ResponseField>

### Success Response

```json
{
  "success": true,
  "dao_pda": "Dao1xgp...ABC",
  "dao_name": "MyDAO",
  "proposer_token_threshold": "1000000000",
  "proposer_holding_period_hours": 24
}
```

### Disabling Threshold

Setting threshold to `null` or `"0"` disables both the threshold and holding period:

```json
{
  "success": true,
  "dao_pda": "Dao1xgp...ABC",
  "dao_name": "MyDAO",
  "proposer_token_threshold": null,
  "proposer_holding_period_hours": null
}
```

## Token Threshold Examples

<Info>
The threshold is specified in **raw token units**, not decimal amounts. For a token with 9 decimals:
</Info>

| Desired Amount | Raw Units (9 decimals) |
| :--- | :--- |
| 1 token | `"1000000000"` |
| 100 tokens | `"100000000000"` |
| 1,000 tokens | `"1000000000000"` |
| 1,000,000 tokens | `"1000000000000000"` |

## Authorization Logic

A wallet can create proposals if **either** condition is met:
1. The wallet is on the proposer whitelist, **OR**
2. The wallet holds at least `threshold` raw units of the DAO's token (averaged over `holding_period_hours` if set)

Setting the threshold to `null` means only whitelisted proposers can create proposals.
