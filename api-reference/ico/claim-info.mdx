## Overview

Retrieves a user's claimable token balance for a specific ICO. This shows how many tokens the user can claim from escrow after the sale is finalized.

<CodeGroup>

```bash curl
curl -X GET "https://api.zcombinator.io/ico/EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v/claim?wallet=9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"
```

```javascript fetch
const tokenAddress = "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v";
const walletAddress = "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM";

const response = await fetch(
  `https://api.zcombinator.io/ico/${tokenAddress}/claim?wallet=${walletAddress}`
);

const claimInfo = await response.json();
```

```python requests
import requests

token_address = "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"
wallet_address = "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"

response = requests.get(
    f'https://api.zcombinator.io/ico/{token_address}/claim',
    params={'wallet': wallet_address}
)

claim_info = response.json()
```

</CodeGroup>

## URL Parameters

<ParamField path="tokenAddress" type="string" required>
  The Solana token mint address of the ICO
</ParamField>

## Query Parameters

<ParamField query="wallet" type="string" required>
  Base58 encoded public key of the wallet to check claim balance for
</ParamField>

## Response

<ResponseField name="tokens_claimable" type="string">
  Total number of tokens available to claim from all purchases
</ResponseField>

<ResponseField name="tokens_claimed" type="string">
  Number of tokens already claimed
</ResponseField>

<ResponseField name="claimed_at" type="string">
  ISO 8601 timestamp of the last claim (if any)
</ResponseField>

<ResponseField name="token_decimals" type="number">
  Number of decimals for the token (default: 9)
</ResponseField>

### Success Response (Has Claimable Tokens)

```json
{
  "tokens_claimable": "25000000000000",
  "tokens_claimed": "0",
  "token_decimals": 9
}
```

### Success Response (After Partial Claim)

```json
{
  "tokens_claimable": "25000000000000",
  "tokens_claimed": "10000000000000",
  "claimed_at": "2024-01-16T14:30:00.000Z",
  "token_decimals": 9
}
```

### Success Response (No Purchases)

```json
{
  "tokens_claimable": "0",
  "tokens_claimed": "0",
  "token_decimals": 9
}
```

## Understanding Claimable Tokens

When users purchase tokens during an ICO:

| Allocation | Percentage | Timing |
|------------|------------|--------|
| Vault | 50% | Immediate on purchase |
| Escrow (Claimable) | 50% | After sale is finalized |

The `tokens_claimable` field represents the escrow portion (50%) that becomes available once the ICO status changes to `finalized`.

## Claim Eligibility

<Note>
Tokens can only be claimed when the ICO sale status is `finalized`. Check the sale status with [Get ICO Sale Info](/api-reference/ico/sale-info) first.
</Note>

### Remaining Balance Calculation

```javascript
const remainingToClaim = BigInt(tokens_claimable) - BigInt(tokens_claimed);
```

## Rate Limiting

This endpoint is subject to rate limiting:
- **30 requests per IP** per 1-minute window
- Returns HTTP 429 when limit exceeded

## Integration Tips

<Tip>
Poll this endpoint to show users their pending claim balance. Update the UI when `tokens_claimable > tokens_claimed` and the sale is finalized.
</Tip>

<Warning>
Always check the sale status before showing the claim button. Claims are blocked during the `active` sale period.
</Warning>

## Token Decimals

The `token_decimals` field helps you display human-readable token amounts:

```javascript
const displayAmount = Number(tokens_claimable) / Math.pow(10, token_decimals);
// e.g., 25000000000000 / 10^9 = 25000 tokens
```
