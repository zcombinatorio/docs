## Overview

Creates a new ICO sale for a token. This generates an escrow account that will hold tokens during the sale and facilitates purchases and claims.

<CodeGroup>

```bash curl
curl -X POST https://api.zcombinator.io/ico/create \
  -H "Content-Type: application/json" \
  -d '{
    "tokenAddress": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
    "creatorWallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "tokenMetadataUrl": "https://arweave.net/abc123",
    "totalTokensForSale": "1000000000000000",
    "tokenPriceSol": "0.00002",
    "vaultTokenAccount": "4kxNBzd7p9qE7V8L...",
    "treasuryWallet": "8mxCBzd7p9qE7V8L...",
    "treasurySolAmount": "1000000000"
  }'
```

## Request Parameters

<ParamField body="tokenAddress" type="string" required>
  The Solana token mint address for the ICO
</ParamField>

<ParamField body="creatorWallet" type="string" required>
  The wallet address of the ICO creator
</ParamField>

<ParamField body="tokenMetadataUrl" type="string" required>
  URL to the token's metadata JSON (e.g., Arweave or IPFS)
</ParamField>

<ParamField body="totalTokensForSale" type="string" required>
  Total number of tokens to sell in the ICO (as string for large numbers)
</ParamField>

<ParamField body="tokenPriceSol" type="string" required>
  Price per token in SOL (decimal string, e.g., "0.00002")
</ParamField>

<ParamField body="vaultTokenAccount" type="string" required>
  Token account where 50% of purchased tokens will be sent
</ParamField>

<ParamField body="treasuryWallet" type="string" required>
  Wallet address to receive SOL from purchases
</ParamField>

<ParamField body="treasurySolAmount" type="string" required>
  Amount of SOL allocated for treasury in lamports
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the ICO was created successfully
</ResponseField>

<ResponseField name="icoSale" type="object">
  The created ICO sale object (see fields in [Get ICO Sale Info](/api-reference/ico/sale-info))
</ResponseField>

## Token Distribution Flow

When an ICO is created, tokens are distributed as follows during purchases:

<Steps>
<Step title="User Purchases Tokens">
User sends SOL to purchase tokens during the active sale period
</Step>

<Step title="50% to Vault">
Half of the purchased tokens are immediately transferred to the vault token account
</Step>

<Step title="50% to Escrow">
The remaining half stays in escrow for the user to claim after the sale is finalized
</Step>
</Steps>

## Rate Limiting

This endpoint is subject to rate limiting:
- **30 requests per IP** per 1-minute window
- Returns HTTP 429 when limit exceeded

## Next Steps

After creating an ICO:

1. **Fund the escrow**: Transfer tokens to the escrow account
2. **Share the ICO**: Users can purchase at [Purchase Prepare](/api-reference/ico/purchase-prepare)
3. **Monitor progress**: Check sale stats with [Get ICO Sale Info](/api-reference/ico/sale-info)
