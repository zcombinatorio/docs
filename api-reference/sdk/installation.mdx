The SDK requires the following peer dependencies:

```bash
npm install @solana/web3.js @coral-xyz/anchor
```

## Verifying Installation

Test your installation:

```typescript
import { futarchy, VaultType } from '@zcomb/programs-sdk';
import { Connection, PublicKey } from '@solana/web3.js';
import { AnchorProvider } from '@coral-xyz/anchor';

// Create a read-only provider
const connection = new Connection('https://api.mainnet-beta.solana.com');
const provider = new AnchorProvider(
  connection,
  {
    publicKey: PublicKey.default,
    signTransaction: async (tx) => tx,
    signAllTransactions: async (txs) => txs,
  } as any,
  { commitment: 'confirmed' }
);

// Initialize client
const client = new futarchy.FutarchyClient(provider);

console.log('SDK initialized successfully!');
console.log('VaultType.Base:', VaultType.Base);
console.log('VaultType.Quote:', VaultType.Quote);
```

## Environment Setup

### For Server-Side (Node.js)

```typescript
import { Keypair, Connection } from '@solana/web3.js';
import { AnchorProvider, Wallet } from '@coral-xyz/anchor';
import { futarchy } from '@zcomb/programs-sdk';

// Load keypair from file
const keypairData = JSON.parse(fs.readFileSync('./wallet.json', 'utf-8'));
const keypair = Keypair.fromSecretKey(Uint8Array.from(keypairData));

const connection = new Connection(process.env.RPC_URL!);
const wallet = new Wallet(keypair);
const provider = new AnchorProvider(connection, wallet, { commitment: 'confirmed' });

const client = new futarchy.FutarchyClient(provider);
```

### For Browser (with Wallet Adapter)

```typescript
import { useWallet, useConnection } from '@solana/wallet-adapter-react';
import { AnchorProvider } from '@coral-xyz/anchor';
import { futarchy } from '@zcomb/programs-sdk';

function useClient() {
  const { connection } = useConnection();
  const wallet = useWallet();

  const provider = new AnchorProvider(
    connection,
    wallet as any,
    { commitment: 'confirmed' }
  );

  return new futarchy.FutarchyClient(provider);
}
```

## Troubleshooting

### Module Resolution Issues

If you encounter module resolution errors:

```json
// package.json
{
  "type": "module"  // or remove for CommonJS
}
```

### BigInt Serialization

The SDK uses BN.js for large numbers. For JSON serialization:

```typescript
import { BN } from '@coral-xyz/anchor';

// Convert BN to string for JSON
const amount = new BN('1000000000');
console.log(JSON.stringify({ amount: amount.toString() }));
```

### RPC Rate Limits

For production use, consider a dedicated RPC provider:
- [Helius](https://helius.xyz)
- [QuickNode](https://quicknode.com)
- [Triton](https://triton.one)
