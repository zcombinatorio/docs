## Overview

Creates a new quantum market proposal under a specified moderator. The creator must be whitelisted or hold sufficient tokens to create proposals.

<CodeGroup>

```bash curl
curl -X POST https://api.zcombinator.io/api/proposals?moderatorId=1 \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "title": "Should we slash 0x123...?",
    "description": "Proposal to slash staker for protocol violation",
    "proposalLength": 86400,
    "creatorWallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "creatorSignature": "5K7x...",
    "attestationMessage": "{\"action\":\"withdraw\",\"poolAddress\":\"ABC123\",\"timestamp\":1735789200,\"nonce\":\"xyz789\"}",
    "markets": 4,
    "market_labels": ["No", "20%", "40%", "60%"]
  }'
```

```python requests
import requests
import json

data = {
    "title": "Should we slash 0x123...?",
    "description": "Proposal to slash staker for protocol violation",
    "proposalLength": 86400,
    "creatorWallet": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "creatorSignature": "5K7x...",
    "attestationMessage": json.dumps({
        "action": "withdraw",
        "poolAddress": "ABC123",
        "timestamp": 1735789200,
        "nonce": "xyz789"
    }),
    "markets": 4,
    "market_labels": ["No", "20%", "40%", "60%"]
}

response = requests.post(
    'https://api.zcombinator.io/api/proposals',
    params={'moderatorId': 1},
    json=data,
    headers={
        'Content-Type': 'application/json',
        'x-api-key': 'YOUR_API_KEY'
    }
)

result = response.json()
```

## Authentication

<ParamField header="x-api-key" type="string" required>
  API key for authentication
</ParamField>

## Query Parameters

<ParamField query="moderatorId" type="string" required>
  The moderator ID to create the proposal under
</ParamField>

## Request Parameters

<ParamField body="title" type="string" required>
  Proposal title displayed in the market interface
</ParamField>

<ParamField body="description" type="string" required>
  Detailed proposal description explaining the context and implications
</ParamField>

<ParamField body="proposalLength" type="number" required>
  Duration of the market in seconds (e.g., 86400 for 24 hours)
</ParamField>

<ParamField body="creatorWallet" type="string" required>
  Solana wallet address of the proposal creator
</ParamField>

<ParamField body="creatorSignature" type="string" required>
  Base58 Ed25519 signature on the attestation message
</ParamField>

<ParamField body="attestationMessage" type="string" required>
  JSON attestation message containing action details. Format:
  ```json
  {
    "action": "withdraw",
    "poolAddress": "string",
    "timestamp": number,
    "nonce": "string"
  }
  ```
</ParamField>

<ParamField body="spotPoolAddress" type="string">
  Meteora pool address (optional)
</ParamField>

<ParamField body="markets" type="number" default="2">
  Number of market options (2-8)
</ParamField>

<ParamField body="market_labels" type="string[]" default='["Fail", "Pass"]'>
  Labels for each market option. Array length must match `markets` count.
</ParamField>

## Response

<ResponseField name="moderatorId" type="number">
  The moderator ID the proposal was created under
</ResponseField>

<ResponseField name="id" type="number">
  Unique proposal identifier
</ResponseField>

<ResponseField name="title" type="string">
  Proposal title
</ResponseField>

<ResponseField name="description" type="string">
  Proposal description
</ResponseField>

<ResponseField name="status" type="string">
  Current status (initially "Pending")
</ResponseField>

<ResponseField name="createdAt" type="number">
  Unix timestamp of creation (milliseconds)
</ResponseField>

<ResponseField name="finalizedAt" type="number | null">
  Unix timestamp of finalization, or null if still active
</ResponseField>

### Success Response

```json
{
  "moderatorId": 1,
  "id": 42,
  "title": "Should we slash 0x123...?",
  "description": "Proposal to slash staker for protocol violation",
  "status": "Pending",
  "createdAt": 1735789200000,
  "finalizedAt": null
}
```

## Error Responses

<AccordionGroup>
  <Accordion title="400 - Missing Required Fields">
    ```json
    {
      "error": "Missing required fields"
    }
    ```

    Ensure all required parameters are provided in the request body.
  </Accordion>

  <Accordion title="400 - Invalid Attestation">
    ```json
    {
      "error": "Invalid attestation message or signature"
    }
    ```

    The signature does not match the attestation message, or the message format is invalid.
  </Accordion>

  <Accordion title="403 - Wallet Not Authorized">
    ```json
    {
      "error": "Wallet not authorized to create proposals"
    }
    ```

    The creator wallet is not whitelisted and does not hold sufficient token balance.
  </Accordion>

  <Accordion title="409 - Active Proposal Exists">
    ```json
    {
      "error": "Active proposal already exists for this pool"
    }
    ```

    Only one active proposal can exist per pool at a time.
  </Accordion>
</AccordionGroup>

## Next Steps

After creating a proposal:

1. **Monitor** the market using [Market Status](/quantum-markets/market-status)
2. **Share** the proposal with your community to drive trading activity
3. **Wait** for the `proposalLength` duration to elapse
4. **Finalize** the market based on the leading TWAP option
